1. 均值回复¶

关键的定量交易概念之一是均值回归。这个过程是指一个时间 显示出回归历史平均值的趋势。当价格很低时， 在期望系列将返回到一个平均值，从而我们进入市场获利。

随机游走（random walk）

随机游走是一个时间序列下一个定向运动完全独立于任何过去的运动。

$dx_t=dW_t$

$dW_t \sim N(0,dt)$

然而，平均回复时间序列不同。下一个时间段的时间序列值的变化正比于当前值。

$dx_t=\theta (\mu-x_t) dt+\sigma dW_t$

Augmented Dickey-Fuller (ADF) Test-检验单位根

$\Delta y_t=\alpha+\beta t+\gamma y_(t-1)+b_1\Delta y_(t-1)+...$

the null hypothesis $\gamma = 0$ against the alternative hypothesis of $\gamma < 0.$

2. 平稳性¶

$E(X_t)=constant$

$Var(X_t)=constant$

$cov(x_t,x_(t+s))=f(s) 与初始时间点无关$

Hurst Exponent- 时间序列的平稳性。

赫斯特指数计算背后的想法是我们可以使用对数价格的方差 系列来评估扩散行为的速度。

$Var(\gamma)=(log(t+\gamma)-log(t))^2$

$(log(t+\gamma)-log(t))^2 \sim \gamma^(2H)$

H <0.5 - 时间序列是平均回复

H = 0.5 - 时间序列是几何布朗运动

H> 0.5 - 时间序列具有趋势

from numpy import cumsum, log, polyfit, sqrt, std, subtract
from numpy.random import randn
def hurst(ts):
    lags = range(2, 100)
    tau = [sqrt(std(subtract(ts[lag:], ts[:-lag]))) for lag in lags]
    # Use a linear fit to estimate the Hurst Exponent
    poly = polyfit(log(lags), log(tau), 1)
    # Return the Hurst exponent from the polyfit output
    return poly[0]*2.0

基本预测手法¶

为了评估分类器的性能，我们可以采用以下两种方法， 即命中率和混乱矩阵。

$hit rate=\frac{1}{n}\sum I(y_i=\hat y_i)$

混淆矩阵通过确定假阳性率来表征这个想法（已知的 统计上作为I型错误）和假阴性率（统计上称为II型错误） 用于监督分类器。 对于二元分类（向上或向下）的情况，我们将有一个2x2 矩阵：

\begin{pmatrix} U_T &U_F \\ D_F &D_T \end{pmatrix}

分类器

  1. logistic regression

假设概率分布为

$P(Y=1|X=x)={\frac {e^{{x'\beta }}}{1+e^{{x'\beta }}}}$, 并且利用最大似然估计最大化联条件率分布解出参数。

(2) Discriminant analysis

在逻辑回归中，我们对给定前一个时间段的概率进行建模 两个滞后回报（P（Y = Uj|L1; L2））作为响应Y的条件分布使用逻辑函数。 在线性判别分析（LDA）中，对Li变量的分布进行建模 是分别基于给定的Y的观测值，并也假设Y的分布。而P（Y = Uj|L1; L2）是通过贝叶斯定理得到的。

$ P（Y = Uj|L1; L2) \propto P (L1; L2|Y = Uj) P(Y = Uj)$

LDA的一个重要数学假设是所有类别（例如“上”和“下”）

(3). 支持向量机

为了训练支持向量机（SVM）， 它通过线性分离边界而分离不同的类，以达到最大化margin的目的，即使所有点尽量远离分类平面。 SVM的主要优势 是它们允许非线性扩大特征空间以包括显着的非线性， 同时仍然保持显着的计算效率，使用称为“内核”的过程。

def create_lagged_series(symbol, st, en, lags=5):
    ts = tu.pro_api().daily(ts_code=symbol, start_date=st, end_date=en).drop(['ts_code'], axis=1).sort_values(by='trade_date')
     # Create the new lagged DataFrame
    tslag = pd.DataFrame(index=ts['trade_date'])
    tslag["Today"] = ts["pct_chg"].tolist()
    tslag["Volume"] = ts["vol"].tolist()
    # Create the shifted lag series of prior trading period close values
    for i in range(0, lags):
        tslag["Lag%s" % str(i+1)] = ts["pct_chg"].shift(1+i).tolist()
        tsret = pd.DataFrame(index=tslag.index)
        tsret["Volume"] = tslag["Volume"]
        tsret["Today"] = tslag["Today"]
    for i,x in enumerate(tsret["Today"]):
        if (abs(x) < 0.0001):
             tsret["Today"][i] = 0.0001
            # Create the lagged percentage returns columns
    for i in range(0, lags):
        tsret["Lag%s" % str(i+1)] = \
        tslag["Lag%s" % str(i+1)]
        # Create the "Direction" column (+1 or -1) indicating an up/down day
    tsret["Direction"] = np.sign(tsret["Today"])
    tsret = tsret[tsret.index >= st]
    tsret=tsret.dropna(axis=0,how='any')
    return tsret

def annualised_sharpe(returns, N=252):
    return np.sqrt(N) * returns.mean() / returns.std()
def stock_sharpe_ratio(symbol,st,en):
    pdf = tu.pro_api().daily(ts_code=symbol, start_date=st, end_date=en).drop(['ts_code'], axis=1).sort_values(by='trade_date')
    # Use the percentage change method to easily calculate daily returns
    pdf['daily_ret'] = pdf['pct_chg']
    pdf['excess_daily_ret'] = pdf['daily_ret'] - 0.05/252
    # Return the annualised Sharpe ratio based on the excess daily returns
    return annualised_sharpe(pdf['excess_daily_ret'])

VaR(Value at Risk),在一定概率水平（置信度）下，组合在未来特定时期内的最大可能损失。

用公式表示为：

　　P(ΔPΔt≤VaR)=a

　　字母含义如下：

　　P——资产价值损失小于可能损失上限的概率。

　　ΔP——某一金融资产在一定持有期Δt的价值损失额。

　　VaR——给定置信水平a下的在险价值，即可能的损失上限。

　　a——给定的置信水平

$\mu + Z_\alpha \sigma$

from scipy.stats import norm
def var_cov_var(P, c, mu, sigma):
    alpha = norm.ppf(1-c, mu, sigma)
    return P - P*(alpha + 1)
